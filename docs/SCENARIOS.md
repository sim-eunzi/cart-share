# 시나리오 케이스 (M1)

유저 유형별 흐름과 예외 케이스 정리. 기획 스펙은 [`SPEC.md`](./SPEC.md) 참고.

## 유저 유형

- **생성자 (나)** — 카트를 만들고 나중에 수정하는 사람. 공유 토큰 + 수정 토큰 둘 다 가짐.
- **친구 (구경꾼)** — 공유 링크만 받은 사람. 읽기만 가능.

---

## A. 생성자 — 카트 만들기

### A-1. 정상 흐름 (Happy path)
1. `/create` 진입
2. 제목 입력 ("여름 원피스 후보")
3. 상품 URL 여러 개 붙여넣기 (줄바꿈으로 구분)
4. "카드 생성" 클릭
5. → 서버가 각 URL OG 추출 (이미지·제목)
6. → 결과 화면에 **공유 링크 + 수정 링크** 표시
7. 두 링크 각각 "복사" 버튼으로 복사

### A-2. 예외 케이스
| 케이스 | 기대 동작 |
|---|---|
| 제목 비움 | 생성 막고 안내, 또는 기본 제목("내 카트") 자동 |
| URL 0개 | 생성 막고 "URL을 1개 이상 넣어주세요" |
| 일부 URL이 OG 추출 실패 | 그 카드는 이미지/제목 없이(placeholder) 생성, 전체는 성공 |
| URL 형식 오류 (http 아님) | 해당 줄만 건너뛰거나 표시, 나머지는 진행 |
| 같은 URL 중복 | 일단 그대로 허용 (M1), 중복 제거는 백로그 |
| 추출이 오래 걸림 | 로딩 표시, URL별 타임아웃 후 실패 폴백 |

---

## B. 친구 — 공유 링크로 구경

### B-1. 정상 흐름
1. 공유 링크(`/c/[shareToken]`) 열기
2. 카드 그리드로 상품들 구경 (이미지·제목·가격)
3. 마음에 드는 카드의 "사러 가기" 클릭
4. → 새 탭으로 원본 쇼핑몰 이동

### B-2. 예외 케이스
| 케이스 | 기대 동작 |
|---|---|
| 잘못된/만료된 공유 토큰 | "없는 카트입니다" 404 화면 |
| 카드 이미지 깨짐 | placeholder 이미지로 대체 |
| 가격 미입력 카드 | 가격 자리 비워두거나 "가격 미정" |
| 친구가 수정 시도 | 불가능 (읽기 전용 UI, 수정 토큰 없음) |
| 빈 카트 (상품 0개) | "아직 담긴 상품이 없어요" 안내 |

---

## C. 생성자 — 나중에 수정

### C-1. 정상 흐름
1. 수정 링크(`/e/[editToken]`) 열기
2. 제목 변경
3. 상품 추가 (URL 입력 → OG 추출 → 카드 추가)
4. 상품 삭제
5. 가격 수동 입력/수정
6. 변경 사항 저장/반영

### C-2. 예외 케이스
| 케이스 | 기대 동작 |
|---|---|
| 잘못된 수정 토큰 | 404 화면 |
| 추가하려는 URL 추출 실패 | placeholder로 추가, 수동 입력 유도 |
| 모든 상품 삭제 | 빈 카트 허용, 공유 링크는 빈 상태로 보임 |
| 제목 빈 값으로 변경 | 막거나 기본값 유지 |
| 수정 링크를 친구에게 잘못 전달 | (주의) 친구가 수정 가능해짐 → UI에서 "이 링크는 나만 보관" 경고 문구 |

---

## D. 토큰/링크 관련 주의 시나리오

| 케이스 | 기대 동작 |
|---|---|
| 공유 페이지 응답에 수정 토큰 노출 | **절대 금지.** API 단에서 제외 (SPEC §6) |
| 생성자가 수정 링크 분실 | M1은 복구 수단 없음 → 결과 화면에서 보관 안내 강조 |
| 공유 토큰으로 수정 URL 추측 시도 | 토큰은 랜덤이라 추측 불가, 권한 분리됨 |

---

## 우선순위 (M1 구현 순서 제안)
1. A-1 생성 happy path + 결과 링크 2개
2. B-1 공유 보기 + 사러 가기
3. C-1 수정 (추가/삭제/제목/가격)
4. 각 화면의 404 · 빈 상태 · 추출 실패 폴백
