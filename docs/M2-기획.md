## 1. M2 목표 (한 줄)

> **"인플루언서가 만든 카드가 인스타/카톡에서 잘 퍼지고, 본 사람이 또 만들게 한다"** — 즉 **바이럴 루프를 완성·강화**한다.

M1은 "만들고 공유된다"까지. M2는 **그 공유가 실제로 퍼지고 새 유저를 데려오는지**를 만든다. 매출(어필리에이트)은 M3로 미루고, M2는 **성장 엔진**에 집중한다.

## 2. 우선순위 근거 (RICE 간소화)

각 후보를 Reach·Impact·Confidence ÷ Effort로 평가. (점수 1~5, Effort 작을수록 유리)

| 후보 기능 | Reach | Impact | Confid. | Effort | 판단 |
|---|---|---|---|---|---|
| 공유 UX 최적화 (인스타/카톡) | 5 | 5 | 4 | 2 | **1순위** — 루프의 입구 |
| "나도 만들기" CTA (루프 닫기) | 5 | 5 | 4 | 1 | **2순위** — K-factor 직결, 공수 적음 |
| 키치 카드 디자인 보완 | 4 | 4 | 3 | 3 | **3순위** — 공유 욕구↑ |
| 클릭 수 측정(가벼운 분석) | 5 | 3 | 5 | 2 | **4순위** — M3 매출의 전제 |
| 카테고리로 묶기 | 3 | 2 | 2 | 3 | **보류** — 루프와 무관 |

> 핵심 논리: M2의 모든 기능은 "공유 → 조회 → 재생성" 루프를 키우는가로 판단. 카테고리 묶기는 정리 편의일 뿐 루프를 안 키우므로 뺀다(필요 신호가 관찰되면 그때).

## 3. M2 범위

### 포함
1. **공유 경험 최적화** — 인스타 스토리/카톡/링크복사 원탭 공유, 공유 미리보기(OG) 다듬기
2. **"나도 만들기" CTA** — 공유 카트 하단에 워터마크 + 만들기 유도
3. **키치 카드 디자인 보완** — 카드 템플릿/스티커, 공유용 이미지 퀄리티
4. **가벼운 클릭 측정** — 카드/상품 클릭 수 집계(노스스타·K-factor 기초)

### 명시적으로 하지 않는 것 (스코프 고정)
- 카테고리로 묶기 (보류)
- 어필리에이트 정산·쿠팡 파트너스 연동 (→ M3)
- 로그인/회원 (여전히 무로그인 + 토큰 유지)
- 인플루언서 대시보드(전환 통계 풀버전) (→ M3)

## 4. 성공 지표 (M2 끝났을 때 이게 보여야 함)

- **노스스타**: 주간 생성·공유된 카드 수 (증가 추세)
- **K-factor**: 공유 카드 1개당 새로 생성된 카드 수 (목표: 측정 가능 상태 + >0)
- Activation: 신규 방문자가 첫 카드 생성→공유까지 도달한 비율
- 공유 버튼 클릭 수 / 공유 카드 조회 수

### #M2-1 [FE] 공유 시트(원탭 공유 UI) — 담당: EUNJI
**목적** 공유가 바이럴 입구이므로 마찰을 0에 가깝게.
**할 일**
- [ ] 생성 완료 화면 & 공유 카트 화면에 "공유하기" 버튼 → 공유 시트
- [ ] 옵션: 링크 복사 / 카카오톡 공유 / 인스타 스토리용 이미지 저장(또는 Web Share API)
- [ ] 모바일에서 OS 기본 공유시트(`navigator.share`) 우선 활용
**완료 기준** 모바일에서 버튼 누르면 카톡/인스타/복사로 한 번에 공유된다.
**의존성** 없음

### #M2-2 [BE/FE] 카톡 공유 + OG 미리보기 정교화 — 담당: 친구(BE) + EUNJI
**목적** 카톡/인스타에 붙였을 때 카드가 예쁘게 미리보기로 뜨게.
**할 일**
- [ ] 카카오 JS SDK 연동(공유 메시지 템플릿: 제목+대표 이미지+링크)
- [ ] 카트별 동적 OG 이미지 점검(대표 상품 썸네일/개수 반영)
- [ ] 카톡 공유용 도메인/앱키 설정
**완료 기준** 카톡에 링크 붙이면 카트 제목·대표 이미지가 카드형으로 뜬다.
**의존성** #M2-1

### #M2-3 [FE/Design] "나도 만들기" CTA + 워터마크 — 담당: EUNJI + designer
**목적** 루프 닫기 — 본 사람을 새 생성자로.
**할 일**
- [ ] 공유 카트(`/c/[id]`) 하단에 "이 카드 왓츠인마이카트로 만듦" + [나도 만들기] 버튼(→ /create)
- [ ] 인스타 스토리용 저장 이미지에도 작은 워터마크
- [ ] CTA 클릭 추적 이벤트
**완료 기준** 공유 카트에서 CTA로 /create 진입이 측정된다.
**의존성** #M2-5(측정)

### #M2-4 [Design/FE] 키치 카드 템플릿·스티커 — 담당: designer + EUNJI
**목적** "공유하고 싶은" 비주얼 = 자생 광고.
**할 일**
- [ ] 픽셀/레트로 컨셉 유지하며 카드 템플릿 2~3종
- [ ] 스티커/이모지 데코 몇 종(선택 적용)
- [ ] 공유 이미지 해상도·여백 정리(인스타 스토리 9:16 대응)
**완료 기준** 사용자가 템플릿을 골라 카드 분위기를 바꿀 수 있고, 저장 이미지가 깔끔하다.
**의존성** 없음 (병렬 가능)

### #M2-5 [BE] 가벼운 클릭/조회 측정 — 담당: 친구(BE)
**목적** 노스스타·K-factor를 보려면 최소한의 집계가 필요.
**할 일**
- [ ] 이벤트 기록: 카트 생성 / 공유 클릭 / 카트 조회 / 상품 클릭 / "나도 만들기" 클릭
- [ ] 카트별 조회수·상품 클릭수 간단 집계 (DB 카운터 또는 경량 분석툴 예: Vercel Analytics/Plausible)
- [ ] (선택) 카트 만든 사람이 자기 카트 조회수 보기
**완료 기준** 주간 생성/공유/조회 수와 "공유→생성" 전환을 숫자로 볼 수 있다.
**의존성** 없음 (먼저 깔면 #M2-3 추적이 쉬움)

### #M2-6 [QA] M2 통합 검수 — 담당: qa
**목적** 데모/실사용 중 안 깨지게.
**할 일**
- [ ] 모바일 실기기에서 공유 플로우 끝까지(카톡/인스타/복사)
- [ ] 공유 카트 → "나도 만들기" → 새 카트 생성까지 루프 검증
- [ ] OG 미리보기가 카톡/슬랙 등에서 실제로 뜨는지
**완료 기준** 핵심 루프가 모바일에서 끊김 없이 동작.
**의존성** #M2-1~5

## 6. 진행 순서

```
측정 세팅(#M2-5) ─┐
공유 시트(#M2-1) ─┼→ 카톡/OG(#M2-2)
                  │
키치 템플릿(#M2-4)─┘   "나도 만들기" CTA(#M2-3)
                          ↓
                   QA 통합 검수(#M2-6) → 배포 → 지표 관찰
```
병렬 포인트: 친구가 #M2-5(측정)·#M2-2(카톡/OG)를 맡는 동안, EUNJI는 #M2-1(공유 시트)·#M2-4(디자인)를 진행. #M2-3은 측정이 깔린 뒤 붙이면 추적까지 한 번에.
